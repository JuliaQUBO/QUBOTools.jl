Basic Usage

using QUBOTools

By loading the package with the using statement, only a few constants will be dumped in the namespace, most of them model types.

Basic File I/O

To read and write QUBO models one is expected to use the Base.read/Base.write API.

# File Path
path = joinpath(@__DIR__, "data", "problem.json")

model = QUBOTools.read_model(path)

Info

The read_model, read_model! and write_model methods will try to infer the file format from the file extension, which can be manually set using an extra optional parameter.

Data Access

When querying a model, one should rely on the provided methods, whose definitions are listed in the API Reference.

QUBOTools.description(model)
"Simple QUBO Problem"
QUBOTools.linear_terms(model)
Dict{Int64, Float64} with 2 entries:
  2 => 0.4
  3 => -4.4
QUBOTools.quadratic_terms(model)
Dict{Tuple{Int64, Int64}, Float64} with 2 entries:
  (1, 2) => -0.8
  (1, 3) => 6.0

File formats

Conversion between formats

One of the main functionalities of this package is to allow fast conversion from a QUBO file format to another. Achieving this is as simple as writing the loaded model but providing a different specification:

QUBOTools.write_model(stdout, model, QUBO())
c scale : 2.7
c offset : 1.93
c id : 2
c description : Simple QUBO Problem
p qubo 0 3 2 2
c linear terms
2 2 0.4
3 3 -4.4
c quadratic terms
1 2 -0.8
1 3 6.0

It can also be used to switch between variable domains while keeping the file format unchanged:

QUBOTools.write_model(stdout, model, BQPJSON(SpinDomain(); indent = 4))
