Models

QUBOTools.AbstractModel — Type
AbstractModel{D<:VariableDomain}

Represents an abstract QUBO Model and should support most of the queries made available by QUBOTools.

Example

A common use case is to build wrappers around the Model concrete type:

struct ModelWrapper{D} <: AbstractModel{D}
    model::Model{D,Int,Float64,Int}
    attrs::Dict{String,Any}
end

QUBOTools.backend(mw::ModelWrapper) = mw.model

As shown in the example above, implementing a method for the backend function gives access to most fallback implementations.

source
QUBOTools.Model — Type
Model{
    D <: VariableDomain,
    V <: Any,
    T <: Real,
    U <: Integer
} <: AbstractModel{D}

The Model was designed to work as a general for all implemented interfaces. It is intended to be the core engine behind the target codecs.

MathOptInterface/JuMP

Both V <: Any and T <: Real parameters exist to support MathOptInterface/JuMP integration. By choosing V = MOI.VariableIndex and T matching Optimizer{T} the hard work should be done.

QUBOTools.density — Function
density(model)::Float64

Computes the density $\rho$ of non-zero terms in a model, according to the expression

\[\rho = \frac{2Q + L}{N^{2}}\]

where $L$ is the number of non-zero linear terms, $Q$ the number of quadratic ones and $N$ the number of variables.

If the model is empty, returns NaN.

source
QUBOTools.linear_density — Function
linear_density(model)::Float64

Computes the linear density $\rho_{l}$, given by

\[\rho_{l} = \frac{L}{N}\]

where $L$ is the number of non-zero linear terms and $N$ the number of variables.

source
QUBOTools.quadratic_density — Function
quadratic_density(model)::Float64

Computes the linear density $\rho_{q}$, given by

\[\rho_{q} = \frac{2Q}{N (N - 1)}\]

where $Q$ is the number of non-zero quadratic terms and $N$ the number of variables.

QUBOTools.adjacency — Function
adjacency(model)::Dict{Int,Set{Int}}
adjacency(G::Vector{Tuple{Int,Int}})::Dict{Int,Set{Int}}
adjacency(G::Set{Tuple{Int,Int}})::Dict{Int,Set{Int}}

Computes the adjacency list representation for the quadratic terms of a given model. A mapping between each variable index and the set of its neighbors is returned.

adjacency(model, k::Integer)::Set{Int}
adjacency(G::Vector{Tuple{Int,Int}}, k::Integer)::Set{Int}
adjacency(G::Set{Tuple{Int,Int}}, k::Integer)::Set{Int}

If a second parameter, an integer, is present, then the set of neighbors of that node is returned.

Warning

Computing specific neighborhoods is expensive. Thus, it is recommended that one stores the adjacency list for repeated access.
